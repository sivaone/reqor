## Successes

- Development completed on time for the final MVP slice.
- Shared export load/redact infrastructure (5.2 → 5.3) with intentional dual redaction dialects.
- Epic-context overrode stale planning: draft-aware `POST` exports shipped; planning’s `GET …/:requestId` never implemented.
- AD-3 held: server-side cURL/snippet serialization only.
- Strong review → patch loop closed most race/stale UI issues before merge.
- Workspace `build` / `test` / `typecheck` stayed green across the epic.

## Challenges

- One post-merge bug: snippet export popover infinite update loop (fixed).
- Recurring async UI class of issues (import close-in-flight, Copy cURL generation, snippet stale tabs) mitigated per-story with generation tokens rather than a shared pattern.
- cURL tokenizer edge cases deferred (quotes, noise tokens, scheme-less URLs).
- No dedicated web component/hook tests for the snippet popover (deferred).

## Key insights

1. “On time” worked because scope stayed narrow and deferrals were honest.
2. Epic-level context correcting planning APIs prevented a draft-awareness miss.
3. The one ship bug was a testing/process gap (popover races mostly manual), not a feature-design failure.
4. Two redaction dialects (`SECRET_MASK` / `••••••` for Copy cURL vs `/* SECRET */` for snippets) are intentional product behavior — keep them documented.
